Please consult the appropriate pages on the FINESCE website in order to understand the complete context of the related FINESCE trials and this DSE.

T&Cs¶

Overview¶

Metering2Orion consists of:
  • a REST service (developed in Java) which allow clients (e.g. DSOs) to register data about metering and load profile in a specific area into an instance of ORION Context Broker GE;
  • a Java client which accepts data about metering and load profile coming from the smart meters (via either an IoT gateway or an existing legacy system) and then “passes” the data to the above mentioned REST service.

Target Usage¶

  • DSOs which need information on metering and load profile in a specific area
  • Third-party applications (such as those developed by SMEs involved in Phase3)

DSE Description¶

Metering2Orion (REST service named "metering-input")¶

IMPORTANT: we kindly remind you that this software module is strictly tight to the use of ORION Context Broker GE thus meaning that you should have previously set up an ORION instance of yours. Please have a look at ORION GE documentation at the following link http://catalogue.fi-ware.org/enablers/publishsubscribe-context-broker-orion-context-broker

Download the artefact (in the form a .war file) from the bottom of the page.
Open the .war file (you can use an appropriate software like “winRar” or "7zip"); open the “config.properties”s file located in the following directory: WEB-INF/classes/ and then amend the following configuration properties “meteringInput.orionServerUrl” and “meteringInput.registerContexPath”:

meteringInput.orionServerUrl = http://(your ORION instance URL):1026
meteringInput.registerContexPath = NGSI10/updateContext

These two extra parametres must be left blank as they are specific of FINESCE WP4 implementation
meteringInput.meterDetailServerUrl =
meteringInput.meterDetailPath =

Save the changes to the .properties file and deploy the .war file on a Tomcat application server using the “Tomcat Web Application Manager” (see attached screenshot). We recommend you to use an updated version of Tomcat; we have used Apache Tomcat 7.xx.


Figure 1: How to the deploy the metering-input .war file

To verify that the deploy of the metering-input application was done successfully and that it consequently runs, you have to type either one of the following URLs in a common browser like Google Chrome or Microsoft Internet Explorer:
  • http://(your tomcat ip)/metering-input/rest/meterDataInput
  • http://(your tomcat ip)/metering-input/rest/loadDataInput

and you should get a page like the following (see attached screenshot).


Figure 2: Test "meterDataInput" GET method in a browser


Figure 3: Test "loadDataInput" GET method in a browser

If either of the previous pages appears, the metering-input application is correctly deployed.

You can now send two different types of "Meter reading" to ORION
  • a "Meter" (an instantaneous measure of power consumed/produced; measurements can be scheduled for example every five minutes)
  • a "Load" (a measure of the energy consumed/produced in a certain time range e.g. every 15 mins)

How to send “Meter reading”(s) (of type "Meter")¶

Now, in order to send a “Meter reading” (of type "Meter") to ORION, use a REST client such as DHC - REST/HTTP API Client chrome extension (see attached screenshot):


Figure 4: Send a "Meter reading" (of type "Meter") to ORION

Please fill in with the following parametres:
SERVICE (POST)
http://(your tomcat ip)/metering-input/rest/meterDataInput

HEADERS
Content-Type: application/xml

BODY
<meter meterId=" (unique identifier of the smart meter for which a measure has been retrieved) "
isConcentrator=" (flag to indicate whether a meter is a concentrator or not) "
upstreamActivePower=" (produced active power in W) "
downstreamActivePower=" (consumed active power in W) "
RP_Q1=" (Q1 reactive power expressed in VAR) "
RP_Q2=" (Q2 reactive power expressed in VAR) "
RP_Q3=" (Q3 reactive power expressed in VAR) "
RP_Q4=" (Q4 reactive power expressed in VAR) " />

and finally click the SEND button.

If you get 200 OK in the HTTP Response, you should have submitted a “Meter reading” (of type Meter) to ORION.

To verify that a new “Meter reading” (of type Meter) has been saved into ORION, an entity of the following type should be created (or updated in case it already exists) in the NoSQL DB underlying ORION (i.e. MongoDB): {

Metering2Orion (Java client named "metering-input-client")¶

Download the artefact (in the form a .jar file) from the bottom of the page.
The metering-input-client module contains a java utility class that has to be referenced as an external library (in a software development java project within a development environment such as Eclipse) in order to let an external application interact with the metering-input REST service described in the previous paragraph.

How to send “Meter reading”(s) (of type "Meter")¶

Before sending the first consumption/production power measure a Meter “entity” needs to be registered by using the registerMeter method. Here is an example of registering a “Meter” entity with meterId equal to "XXXmeterId":

MeteringProxy meteringProxy = new MeteringProxyImpl();
        Meter meter = new Meter("XXXmeterId", false);
        meteringProxy.registerMeter(meter);

Hereafter there is a code snippet which shows how to send a meter reading (of type "Meter") for the “Meter” entity that has been previously registered:

Meter meter = new Meter(
        "XXXmeterId",                // smart meter unique identifier
        false,                                    // this meter is not a concentrator
        System.currentTimeMillis() / 1000,    // timestamp of the moment when data are sent to the REST Service    
        12.3,                    // upstreamActivePower
        123,                     // downstreamActivePower
        456,                    // reactivePowerQ1
        789,                             // reactivePowerQ2
        123,                    // reactivePowerQ3
        45);                            // reactivePowerQ4
Response response = meteringProxy.sendMeterData(meter);
        if(response.getStatus == 200)
            System.out.println(“Everything is fine!”);

How to send “Meter reading”(s) (of type "Load")¶

Before sending the first cumulative load data sample, a Load “entity” needs to be registered by using the registerLoad method. Here is an example of registering a “Load” entity with meterId equal to "XXXmeterId":

MeteringProxy meteringProxy = new MeteringProxyImpl();
        Load load = new Load("XXXmeterId", false);
        meteringProxy.registerLoad(load);

Hereafter there is a code snippet which shows how to send a meter reading (of type "Load"), a load sample, for the “Load” entity that has been previously registered:

meteringProxy = new MeteringProxyImpl();
        Load load = null;
        load=new Load("55555555555",false);
        load.setSampleNumber(5);
        load.setDownstreamActivePowerEEA(1000.0); //consumed power
        load.setUpstreamActivePowerEUA(10.0); //produced power
        load.setIntegrationPeriodRef("Q");
        load.setLoadSampleDate(Calendar.getInstance().getTime());
        load.setReactiveCapacitivePowerEEC(10.0);
        load.setReactiveCapacitivePowerEUC(0.0);
        load.setReactiveInductivePowerEEI(0.0);
        load.setReactiveInductivePowerEUI(0.0);
        load.setTariffType("T1");
        load.setCurrentTime(System.currentTimeMillis() / 1000);
Response response = meteringProxy.sendLoadData(load);
        if(response.getStatus == 200)
            System.out.println(“Everything is fine!”);
