Preface¶

Please consult the appropriate pages on the FINESCE website in order to understand the complete context of the related FINESCE trials and this DSE.

T&Cs¶

Overview¶

WeaFor2Orion consists of:
  • a REST service (developed in Java) which allow clients (e.g. Weather Condition Information Providers) to register data about weather conditions and predictions in a specific area into an instance of ORION Context Broker GE;
  • a Java TIMER client which retrieves (every five minutes) weather conditions and predictions from an external provider (https://developer.forecast.io/ in our implementation) and then “passes” the data to the above mentioned REST service.

Target Usage¶

  • Weather Condition Information Providers (which may be appointed by energy companies that would need information on weather conditions and forecasts in a specific area)
  • Third-party applications (such as those developed by SMEs involved in Phase3)

DSE Description¶

WeaFor2OrionRest (REST service)¶

IMPORTANT: we kindly remind you that this software module is strictly tight to the use of ORION Context Broker GE thus meaning that you should have previously set up an ORION instance of yours. Please have a look at ORION GE documentation at the following link http://catalogue.fi-ware.org/enablers/publishsubscribe-context-broker-orion-context-broker

Download the artefact (in the form a .war file) from the bottom of the page.
Open the .war file (you can use an appropriate software like “winRar” or "7zip"); open the “weafor2orionrest.properties”s file located in the following directory: WEB-INF/classes/ and then amend the following configuration properties:

weafor2orionrest.orionServerUrl = http://(your ORION instance URL):1026
weafor2orionrest.registerContexPath = NGSI10/updateContext

Save the changes to the .properties file and deploy the .war file on a Tomcat application server using the “Tomcat Web Application Manager” (see attached screenshot). We recommend you to use an updated version of Tomcat; we have used Apache Tomcat 7.xx.


Figure 1: How to the deploy the weafor2orionrest.war file

To verify that the deploy of the weafor2orion REST application was done successfully and that it consequently runs, you have to type the following URL in a common browser like Google Chrome or Microsoft Internet Explorer:
  • http://(your tomcat ip)/weafor2orionrest/rest/sendWeatherData

and you should get a page like the following (see attached screenshot).


Figure 2: Test "sendWeatherData" GET method in a browser

If the previous pages appears, the weafor2orion REST application is correctly deployed.

Once retrieved weather forecasts information from a provider of your choice, you can now send it to ORION

WeaFor2Orion (Java TIMER client)¶

Download the artefact (in the form a .jar file) from the bottom of the page.
Open the .war file (you can use an appropriate software like “winRar” or "7zip"); open the “weafor2orionrest.properties”s file located in the following directory: WEB-INF/classes/ and then amend the following configuration properties:

weafor2oriontimer.orionServerUrl = http://(your ORION instance URL):1026
weafor2oriontimer.weatherServerUrl = :8080/weafor2orionrest/rest)
weafor2oriontimer.serviceKey = (your forecast.io service key)

Install a java JDK (e.g. 1.6 / 1.7). The weafor2orion module contains a java timer class that has to be launched from the command line by using the following:

java -jar weafor2orion-0.0.1-SNAPSHOT.jar

It retrieves weather forecasts from an external service provider (in our case http://developer.forecast.io), parses the data and then submit them to your instance of ORION.
