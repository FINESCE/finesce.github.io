Catalogue of FINESCE "Domain Specific Enablers"

There are two options for making DSEs available:

  • Option 1: the specification of the DSE is made publically available, royalty free.
  • Option 2: the specification of the DSE is made publically available, royalty free and the code is available Open Source.
WP DSE Name DSE Description Option

1

Integration kit DSE

Documented process including some examples for the established integrations within WP1

1

2

3

Modbus Connector (ModConn) DSE

Gateway to a local shop floor network

1

3

OBDC Event Sink (EvSi) DSE

Local storage of shop floor events

2

3

Production Planning and Control System Integrator (PPSI) DSE

Connecting factory planning systems to the cloud

1

4

Contract Tailor Processor

Available Q1 2015

This component will be responsible to calculate a new contract for a specific customer based on an incentive plan (issued by a Retailer and approved by a Market Regulator Authority).

2

4

Contract Processor

Available Q1 2015

This component will be responsible to store all the contracts, their validation, and associated user profiles. It could involve the support of operations like contract update/modification (by a Retailer) and contract approval (by end customers).

2

4

Social Events Interface (Social2Orion **)

Social2Orion is a REST service (developed in Java) which allows clients (e.g. Social Events Information Providers) to register data about social events (affecting consumption/production in a specific area such as a concert or a football match) in an instance of ORION Context Broker GE

2

2

Weather Condition Interface (WeaFor2Orion **)

WeaFor2Orion consists of: a REST service (developed in Java) which allow clients (e.g. Weather Condition Information Providers) to register data about weather conditions and predictions in a specific area onto an instance of ORION Context Broker GE + a Java client which retrieves weather conditions and predictions from an external service (forecast.io in our implementation) and then “passes” the data to the above mentioned REST service.

2

4

Contract Information (ContractInformation2Orion **)

ContractInformation2Orion is a REST service (developed in Java) which allows clients (e.g. Retailers) to register data about cost of energy produced from the DERs, costs of transmission system and power plants, energy costs in an instance of ORION Context Broker GE.

2

4

Regulation Compliance Rule Engine

Available Q1 2015

This component will be responsible for checking and confirming the compliance of an incentive plan (issued by a Retailer) to established regulations and rules.

2

4

Incentive Processor

Available Q1 2015

This component will be responsible to calculate a new incentive plan for a specific customer based on an issue resolution plan plan (issued by an Aggregator and approved by a DSO).

2

4

Metering (Metering2Orion **)

Metering2Orion consists of: a REST service (developed in Java) which allow clients (e.g. DSOs) to register data about metering and load profile in a specific area into an instance of ORION Context Broker GE + a Java client which accepts data about metering and load profile coming from the smart meters (via either an IoT gateway or an existing legacy system) and then “passes” the data to the above mentioned REST service.

2

4

Protocol Adapter AMM (Sensor2AMI **)

Available Q1 2015

This component will retrieve DLMS/COSEM data from meters installed at the trial site (metering and load profile) and then feeds an instance of IDAS (Backend) Device Management GE with the data (which, on its turn, forwards data to an instance of ORION Context Broker GE

2

4

Issue Detector Processor (Cosmos2SCILAB + INP-SCILAB + SCILAB2Orion **)

Available Q1 2015

Detects issues related to power losses and voltage drops (in the lines of a smart LV grid) and then sends it to an instance of ORION Context Broker GE

2

5

Charging Optimisation System (COS) DSE

Optimisation of Electric Vehicle Charging

1

6

FINESCE API Mediator (FAM) DSE

Available Q1 2015

It provides a single point-of-entrance to the FINESCE trial infrastructures, exposing the FINESCE API specification. It serves FINESCE-API compatible incoming requests from interested parties, abstracting the heterogeneity of the underlying FINESCE supporting services.

 2

** name of the software module(s) in the WP4 eMarketplace infrastucture implementing the DSE functionality
