Preface¶

Within this document you find a self-contained open specification of the aforementioned Domain Specific Enabler (DSE).

Please consult the appropriate pages on the FINESCE website in order to understand the complete context of the related FINESCE trials and this DSE.

T&Cs¶

Overview¶

ContractInformation2Orion is a REST service (developed in Java) which allows clients (e.g. Retailers) to register data about cost of energy produced from the DERs, costs of transmission system and power plants, energy costs into an instance of ORION Context Broker GE.

This component can be eventually re-used and customized to fulfill the general need of having a REST service which is connected to the ORION GE.

Target Usage¶

  • Retailers (to upload information on cost of energy produced from the DERs, costs of transmission system and power plants, energy costs)
  • Third-party applications (such as those developed by SMEs involved in Phase3)

DSE Description¶

IMPORTANT: we kindly remind you that this software module is strictly tight to the use of ORION Context Broker GE thus meaning that you should have previously set up an ORION instance of yours. Please have a look at ORION GE documentation at the following link http://catalogue.fi-ware.org/enablers/publishsubscribe-context-broker-orion-context-broker

Download the artefact (in the form a .war file) from the bottom of the page.
Open the .war file (you can use an appropriate software like “winRar”), open the “contractinformation2orion.properties”s file located in the following directory: WEB-INF/classes/ and then amend the following configuration properties “contractinformation2orion.orionServerUrl” and “contractinformation2orion.registerContexPath”:

contractinformation2orion.orionServerUrl = http://(your ORION instance URL):1026
contractinformation2orion.registerContexPath = NGSI10/updateContext

Save the changes to the .properties file and deploy the .war file on a Tomcat application server using the “Tomcat Web Application Manager” (see attached screenshot). We recommend you to use an updated version of Tomcat; we have used Apache Tomcat 7.xx.


Figure 1: "How to the deploy the contractinformation2orion .war file

To verify that the deploy of the contractinformation2orion application was done successfully and that it consequently runs, you have to type the following URL in a common browser like Google Chrome or Microsoft Internet Explorer : http://(your tomcat ip)/contractinformation2orion/rest/sendContractInformationData and you should get a page like the following (see attached screenshot).


Figure 2: Test "sendContractInformationData" GET method in a browser

If the previous page appears, the contractinfromation2orion application is correctly deployed.

Now, in order to send a “Contract Information Event” to ORION, use a REST client such as DHC - REST/HTTP API Client chrome extension (see attached screenshot):


Figure 3: Send a "Contract Information Event" to ORION

Please fill in with the following parametres:
SERVICE (POST)
http://(your ip)/social2orion/rest/sendContractInformationData

HEADERS
Content-Type: application/xml

BODY
<contractInformation
contractInformationID=" (a unique identifier of a set of cost information) "
costOfEnergyProducedByDERS=" (cost of energy produced by DERS) "
costOfSystemTransmissionPowerPlants=" (cost of system transmission power plants) "
energyCost=" (cost of energy) "
validityStartDate=" (date from when cost information are valid) "
validityEndDate=" (date till when cost information are valid) "/>

and finally click the SEND button.

If you get 200 OK in the HTTP Response, you should have submitted a “contract information event” to ORION.

To verify that a new “contract information event” has been saved into ORION, an entity of the following type should be created (or updated in case it already exists) in the NoSQL DB underlying ORION (i.e. MongoDB): {
"_id" : {
"id" : "contractinformation2orion",
"type" : "Contractinformation"
},
"attrs" : [{
"name" : "currentTime",
"type" : "long",
"value" : " (system date at the moment the data are received by the REST Service) ",
"creDate" : (date of attribute creation),
"modDate" : (date of attribute last update)
}, {
"name" : "costOfEnergyProducedByDERS",
"type" : "currency",
"value" : " (cost of energy produced by DERS) ",
"creDate" : (date of attribute creation),
"modDate" : (date of attribute last update)
}, {
"name" : "costOfSystemTransmissionPowerPlants",
"type" : "currency",
"value" : " (cost of energy produced by DERS) ",
"creDate" : (date of attribute creation),
"modDate" : (date of attribute last update)
}, {
"name" : "energyCost",
"type" : "currency",
"value" : " (cost of energy) ",
"creDate" : (date of attribute creation),
"modDate" : (date of attribute last update)
}, {
"name" : "validityStartDate",
"type" : "long",
"value" : " (date from when cost information are valid) ",
"creDate" : (date of attribute creation),
"modDate" : (date of attribute last update)
}, {
"name" : "validityEndDate",
"type" : "long",
"value" : " (date till when cost information are valid) ",
"creDate" : (date of attribute creation),
"modDate" : (date of attribute last update)
}],
"creDate" : (date of entity creation),
"modDate" : (date of entity last update)
}
